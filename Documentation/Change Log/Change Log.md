## What Goes Where

### Main Change Log (this file)
- **Major roadmap phases** - Brief entries when complete
- **Version releases** - Significant releases with key highlights
- **Major architectural changes** - High-level summary only

### Monthly Change Logs
- **Detailed daily/weekly entries** - All minor changes, fixes, updates
- **Development progress** - Day-to-day work and incremental improvements
- **Small fixes and tweaks** - Any change that doesn't warrant a version bump

### Completed Items Files (separate files)
- **Completed Bugs.md** - Full details of fixed bugs (moved from Bugs/Bugs.md)
- **Completed Features.md** - Small features completed (moved from Roadmaps/Roadmap-Small Features.md)
- **Completed Improvements.md** - Improvements completed (moved from Roadmaps/Roadmap-Improvements.md)

---

## Monthly Split Instructions

When a monthly change log reaches ~50 entries:
1. Create a new file: `Change Log - [Month] [Year] - 2.md`
2. Update this index to link to both files
3. Continue adding to the newest monthly file

Example structure:
- [December 2025](Change%20Log%20-%20December%202025.md)
- [December 2025 (cont.)](Change%20Log%20-%20December%202025%20-%202.md)

## Format Guide

### Version Entry Format
```markdown
### Version X.Y.Z - Month DD, YYYY
- Major feature/roadmap completion (brief)
- Key highlights (2-3 bullet points max)
```

### Monthly Entry Format
```markdown
#### December 9, 2025
- Fixed: [Description of fix]
- Added: [Description of addition]
- Updated: [Description of update]
- Improved: [Description of improvement]
```
